# Contributing to MxJSON / 贡献指南

感谢您对 MxJSON 项目的关注！我们欢迎各种形式的贡献。

## 如何贡献

### 1. 报告问题 (Issues)

如果您发现 bug 或有功能建议，请创建 Issue：
- 使用清晰的标题描述问题
- 提供复现步骤（如果是 bug）
- 附上相关的错误信息或截图

### 2. 提交代码 (Pull Requests)

1. Fork 本仓库
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 创建 Pull Request

### 3. 贡献能力包 (MXP Packages)

我们特别欢迎贡献实用的 MxJSON 能力包：
- 将您的能力包放在 `examples/` 目录下
- 包含完整的 README 说明
- 提供测试用例

## 代码规范

- 使用 4 空格缩进
- 变量命名使用 camelCase
- 常量命名使用 UPPER_SNAKE_CASE
- 添加必要的注释

## 协议

贡献的代码将采用与本项目相同的 MIT License。

## 联系我们

如有任何问题，请通过 Issue 或邮件联系我们。
